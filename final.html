Passing Parameters to Functions

First things first: I'm approaching this lesson with the understanding that the reader has some basic familiarity with html, css and javascript elements. This tutorial doesn't require any ADVANCED knowledge but I feel like the assignment will be long enough without having to go through setting up a webpage and creating a script. JUST in case, help with site and script creation can be found from Mozilla using these startup files.

My tutorial covers the basic concepts of passing parameters to functions. Entering this class as a programming student (Using primarily Java, Python and C#) with no prior experience with Javascript, I thought I'd cover a topic that's relevant to Javascript as well as the other languages I've studied. I chose this topic specifically because it took me quite a while to understand when I was first introduced to programming. I don't really know why it took so long to wrap my head around the concept of passing parameters, but once it "clicked", it made a number of things significantly easier. As a happy side effect, it forced me to me think about the functions or methods I was writing (or planning to write), knowing that I might be able to reuse a method that could perform an operation on a different parameter (or object!) without having to rewrite that block of code.

Javascript makes use of functions that accept parameters to act on, just as Java, Python, C# or any number of other programming language might. Some key differences with Javascript are variable declarations and how we pass parameters. JS is smart enough to determine the variable type so there's no need to declare integers, strings or decimals types, which is nice. Javascript only passes parameters by value, rather than by reference, but since JS doesn't make me choose one over the other, I won't be boring you with that electrifying lecture. Passing parameters is one of the slickest concepts of programming that I've learned so far, and it's really easy with Javascript - no mussing with data types and math related conflicts or other variable-type related headaches. There's something about writing a function that I've used multiple times for multiple parameters (but only had to write once) that does wonders for my programming confidence, so this is the backbone of my tutorial. Hopefully by the time you've finished reading, everything I've said up until now will make sense.

What do you mean, passing parameters?

When passing a parameter or object, you're handing the function a variable that will be used in whatever instructions have been written to the function. For example, let's say we have a function that accepts any number and adds 5 to it. It might look something like this:


	  var someNumber = 10;
	  addFiveFunction(someNumber);
	  
	  function addFiveFunction(anyNumber) {
		var answer = 5 + anyNumber;
		console.log("The result of the function is " + answer + ".");
		}
		
	  // Value of the calculation is 15;
  1. In the code above, we've assigned a value of 10 to the variable someNumber. We could've assigned that value through any number of ways such as a text field, a selector or a radio menu, but that doesn't matter right now - plus I'll do a small bit of that later. What's important is that the variable has a value of 10. 

    var someNumber = 10;

  2. The addFiveFunction function is then called, and the someNumber variable is given as a parameter of the function (if it's in parentheses, it's a parameter of the function). 

    addFiveFunction(someNumber);

  3. This is the first part of this whole concept that tripped me up at first. Now we have the function definition: 

    function addFiveFunction(anyNumber) {

    The anyNumber portion of this definition is not asking for a variable named anyNumber. This is a definition of a variable to be used in the function below this definition (within the curly braces). Basically, you're taking the value of someNumber and using it in the function called in step 2. To do this, the variable is assigned its own name (anyNumber) and memory space so it doesn't change the value of the original variable.

    Distilled down: 

    addFiveFunction(someNumber);

    Says: "call the function addFiveFunction using the value of someNumber."

    To which the function 

    function addFiveFunction(anyNumber) {

    Responds with: "Sure, I'll answer your call and use your someNumber value, but I'm going to call it anyNumber regardless of what you named it....

  4. What's left is the basic math performed in the function. We've defined a variable answer and assigned it the value of 5 + the value of anyNumber, which we determined in step 3 was actually just the value of someNumber. 

    var answer = 5 + anyNumber;

  5. Obviously, 10 + 5 = 15, and if you were to write this function into your script file and open the debugging console, it would output the message "The result of the function is 15." In fact, you can open the console and see that it does exactly that - I have that portion of the script running on this page. 

    console.log("The result of the function is " + answer + ".");

Ok, but why would I need to do this?

I'd argue the most significant reason is flexibility (or perhaps, efficiency, depending on how you're looking at it). Passing objects and parameters allows us to manipulate all or part of (some data) and reuse it elsewhere. It helps us to write code that's easy to work through and troubleshoot because we can create specific methods that perform specific jobs and pass data between those methods.

In a real life practical application, we might not have a function that adds a preset number (5) to another preset number (10), so the first example is more demonstrative than anything. However, consider an e-commerce website or a point-of-sale terminal that calculates a subtotal and tax. Depending on where you are in the world you are and how much you spend, each of those values will change between transactions and the function has to be able to calculate variable values. Take a look at the form below. Enter any numeric value in the Subtotal box then click the calculate button. The grand total (after adding a 7.8% sales tax) is displayed:

Sifting through the code below, we can see the variables and functions involved with making our tax calculation. We could make one big function to handle every step in this calculation, but having separate functions that perform their own tasks helps with identifying problem areas within the code - if some part of the program doesn't work, this can help to easily identify where the conflict might be hiding. It also operates on a programming concept called encapsulation, or information-hiding. The user really only interacts with one of the functions, but that function might call another function which may call yet another function, and so on. Writing code in this way gets us into the practice of hiding information that doesn't need to readily available to each part of our program or script. Passing variables and functions between the inaccessible portions of our code is mandatory for effective encapsulation. Anyway, take a look:

	
	var taxRate = 1.078;
	var subTotalField = document.getElementById("subTotal");
	var grandTotalField = document.getElementById("grandTotal");
	var invalidSubTotalError = "Please enter a valid amount -  up to 2 decimal places.";
	  
	function displayTotal() {
		var subTotalValue = subTotalField.value;
			if (!subTotalValue || !validNumField(subTotalValue)){
				alert (invalidSubTotalError);
			} else {
				grandTotalField.value = "$" + calcTax(subTotalValue);
			}
	}
	
	function calcTax(subtotal) {
		var grandTotal = (subtotal * taxRate).toFixed(2);
		return grandTotal;
	}
	
	function validNumField(formField) {
		var filter = /^[0-9]+$/i;

		if (filter.test(formField)) {
			return true;
		} else {
			return false;
		}
	}

We have some variable definitions for specific page elements, a variable for the tax rate multiplier, an error message, and a few functions. There's an event listener function for the button, but that isn't listed, but it's important to note that the calculate button calls the displayTotal() function.

I hid some of the styling code in the if/else statement in the displayTotal() function, as it's relevant only to the error handling for that function and isn't specifically necessary for this tutorial. We really only care about these two lines for now:

  1. if (!subTotalValue || !validNumField(subTotalValue)){

    2. and:

  2. alert (invalidSubTotalError);

In the first line, the if statement passes the subTotalValue value to the validNumField() function. This checks to ensure the value is a numeric value and has no more than 2 digits after the decimal point. The second line shows what happens when the alert() function is triggered. If the value does not meet the criteria of the validNumField() function, an alert window pops up displaying the invalidSubTotalError string that was passed as a parameter of the built-in alert() function of Javascript.

Moving on to the else portion of the statement, we pass a third item - this time, we pass a parameter to a function that will retrieve the desired output directly:

grandTotalField.value = "$" + calcTax(subTotalValue);

In the snippet of code above, we're attempting to make the grandTotalField display a dollar sign immediately followed by whatever value is returned from the calcTax() function, using the subTotalValue value as a parameter of that function. Similar to the first example of the tutorial with the addFiveFunction(), the calcTax() function expects a variable to be passed to it so it can use it in its own calculation:

function calcTax(subtotal) {
		var grandTotal = (subtotal * taxRate).toFixed(2);
		return grandTotal;
	}

The calcTax() function renames this passed variable value to subtotal, then declares a new variable named grandTotal. (Note: renaming the passed parameter is not necessary, I do this to illustrate which variable/parameter was sent from the original function call and how the method is using that parameter). The value of the new variable is set to the rounded product of subtotal (the original value passed) and taxRate, the global variable we declared in the first line of the example code. Finally, the function returns the value of grandTotal to the original location it was called:

grandTotalField.value = "$" + calcTax(subTotalValue);}

If you enter a value of 10, this tax calculation comes out to 10.78, which effectively changes the display on screen to $10.78 - just like if it had been written as:

grandTotalField.value = "$" + "10.78";

Final thoughts:

As you can see, being able to pass parameters or objects from function to function is not only an effective way to write code that's easier to debug, but it compartmentalizes your code in a manner that allows you to reuse a function in another, separate part of the program. While this tax calculation function is very narrow in scope, I could reuse the calcTax() method any time in any other method because I'm able to pass any numeric parameter to it.

Consider the error message alert when entering a non-numeric value into the subTotal field. I could modify the code slightly to custom tailor error messages for specific types of errors, pass that message string to another method which would change the alert() message to the specific message or messages that were relevant to my errors.

Yet another example: While the tax rate is a declared constant within the program, adding another text field, check box, radio menu or selector could potentially allow me to change the working tax rate, and I'd simply call a method which retrieves the chosen tax rate and passes it to the calcTax() method as another, separate value.

All of these changes are only be possible because of our ability to pass parameters. It takes a lot of the tedium out of necessary programming, and makes basic processing of information that much easier.
