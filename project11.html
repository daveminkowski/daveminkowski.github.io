Project 11 - Ajax requests

Identify data provided by an Ajax service that you'd like to include in your personal website. You should choose a web service other than those used in the chapter and the Hands-on Projects. If you have an idea for data you'd like to access but are unsure which service might provide that data, perform a web search on a description of the data plus "API." For instance, if you were looking for a source of tide tables, you might search for "tide tables API." Use the documentation for the web service to construct an Ajax request and to display selected data from the service on your website. Note that if you don't have experience with writing PHP, you may need to examine a number of potential APIs to identify one that allows JSON-P or CORS requests, which don't require you to run a proxy.

Weather Report by ZIP code

Using Openweathermap.org's API, a ZIP code can be entered into the search box and a brief report on the area weather is displayed. The JSON object returns quite a few values, but I'm using temperature, humidity, coordinates, a decription of the weather and an icon ID to represent that weather description.

After you enter a zip code and click the button, the JSON response objects and properties will populate a field below this paragraph. The objects and their properties are accessed using their literal names (main.temp or coord.lat). Note the "weather" property is an array and its properties must be accessed as properties of an array element (weather[0].main or weather[0].icon). There's a lot of room for improvement (as far as visually presenting this data) here on the website but I actually took this API and am now working on integrating it into a smart mirror project I started a bit ago, so... a lot of my effort has been spent on that, but these are the basics that I started with. The ZIP field allows only a 5 digit zip, so you can really only use this to find weather in US cities. That's not an issue with the API though, that was my choice. The search parameter can be modified to use coordinates or city codes if you wanted. Lastly, if your request doesn't immediately populate, give it a couple seconds - it's not always instant.
