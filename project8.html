Project 8 - Saving list items as an array of strings

Enhance the feedback form in your project to enable users to choose one or more options from a list of at least five options. Include code that adds user selections to either an array or an object, and ensure that if a user deselects one of the options, it is removed from the array or object. Add code to convert the array or object to a string.

Video Game Hero Ranker

Choose any of the 10 video game protagonists list below, ordered randomly on page load. Selecting a character and clicking the "add" button will rank them in the order you choose - the ranked list will appear below the selection box once you make your first choice. Click the appropriate remove button to delete the character from the list and remove its value from the array.


 

Additionally:

  1. This script takes the value from the selector and puts it into an array when the "add" button is clicked. Whenever a name is added or removed, the list is deleted then recreated based on the existing elements in the array, giving the appearance of being refreshed. Each button is created on the fly as the name is selected form the list and functions as a way to remove the name from the list and from the array (which will then refresh the visual list). Just for fluff, the names in the selector element are randomized on page load.
  2. The "Array Values" display is just a visual representation of each array element being added or removed from the array. The functionality is the same as adding to the ordered list, but I think it helps to show how the array is actually reordered rather than just displaying elements of an array that could potentially contain empty elements after they've been removed from the list.
  3. The other thing I'd want to fix is to disable each name in the selector once it was chosen then re-enable it once it was removed from the ranked list. It'd defintely help the user see which options weren't already chosen and the project feels incomplete without having that "feedback" from the list. Being able to add the same value several times consecutively also makes for some weirdness when trying to remove that name. (Try adding the same name 5-10 times consecutively, then just click to remove any one of them to see what I mean) I probably could have handled with checkboxes and an onChange handler, but I didn't like that option, visually.
